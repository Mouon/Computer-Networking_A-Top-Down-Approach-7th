# 4.1 네트워크 계층 개요

4장에서는 **네트워크 계층(network layer)** 을 다룬다.

앞 장의 트랜스포트 계층은 서로 다른 호스트에서 동작하는 프로세스 사이의 논리적 통신을 제공했다.

그런데 트랜스포트 계층의 세그먼트가 실제로 목적지 호스트까지 가려면, 중간의 라우터들을 지나 네트워크를 이동해야 한다.

이 일을 담당하는 계층이 네트워크 계층이다.

네트워크 계층은 송신 호스트에서 수신 호스트까지 **데이터그램(datagram)** 을 전달한다.

## 4.1.1 네트워크 계층의 역할

네트워크 계층의 핵심 역할은 다음 두 가지이다.

1. **포워딩(forwarding)**
2. **라우팅(routing)**

두 단어가 비슷해 보이지만 의미는 다르다.

### 포워딩

포워딩은 라우터의 입력 링크로 들어온 패킷을 적절한 출력 링크로 이동시키는 작업이다.

즉, 라우터 내부에서 일어나는 지역적인 동작이다.

예를 들어 어떤 패킷이 라우터 R1에 도착했다고 하자.

라우터 R1은 패킷의 목적지 주소를 보고 자신의 포워딩 테이블을 확인한다.

그 결과 이 패킷을 2번 출력 포트로 보내야 한다면, 라우터는 패킷을 2번 출력 포트로 넘긴다.

이것이 포워딩이다.

### 라우팅

라우팅은 송신 호스트에서 수신 호스트까지 패킷이 어떤 경로를 따라갈지 결정하는 작업이다.

라우팅은 네트워크 전체 관점의 문제이다.

라우터들은 라우팅 알고리즘을 통해 경로 정보를 계산하고, 그 결과를 바탕으로 포워딩 테이블을 만든다.

예를 들어 H1에서 H2로 가는 가장 좋은 경로가 R1 -> R3 -> R5라면, 각 라우터는 이 경로에 맞게 패킷을 다음 라우터로 넘길 수 있어야 한다.

정리하면 다음과 같다.

| 구분 | 의미 | 관점 |
| --- | --- | --- |
| 포워딩 | 입력 링크의 패킷을 출력 링크로 이동 | 라우터 내부 |
| 라우팅 | 출발지에서 목적지까지의 경로 결정 | 네트워크 전체 |

## 4.1.2 데이터 평면과 제어 평면

네트워크 계층은 크게 두 부분으로 나눠 볼 수 있다.

1. **데이터 평면(data plane)**
2. **제어 평면(control plane)**

### 데이터 평면

데이터 평면은 라우터별로 동작하는 기능이다.

패킷이 라우터에 도착했을 때, 라우터가 그 패킷을 어느 출력 포트로 보낼지 결정하고 실제로 넘기는 작업이 데이터 평면에 속한다.

즉, 데이터 평면의 핵심은 포워딩이다.

데이터 평면은 매우 빠르게 동작해야 한다.

라우터는 초당 엄청난 수의 패킷을 처리해야 하므로, 포워딩 동작은 하드웨어 수준에서 빠르게 수행되는 경우가 많다.

### 제어 평면

제어 평면은 네트워크 전체에서 경로를 계산하고, 포워딩 테이블을 구성하는 기능이다.

즉, 제어 평면의 핵심은 라우팅이다.

제어 평면은 데이터 평면보다 더 넓은 시야를 가진다.

패킷 하나하나를 빠르게 넘기는 것보다, 네트워크 상태를 보고 경로를 결정하는 역할에 가깝다.

이 책에서는 4장에서 데이터 평면을 먼저 다루고, 5장에서 제어 평면을 자세히 다룬다.

## 4.1.3 전통적인 라우터 기반 제어 평면

전통적인 인터넷에서는 각 라우터 안에 포워딩 기능과 라우팅 기능이 함께 들어 있다.

라우터는 다음 두 가지를 모두 수행한다.

- 들어온 패킷을 출력 포트로 넘김
- 라우팅 프로토콜을 실행해 포워딩 테이블을 계산함

예를 들어 OSPF나 BGP 같은 라우팅 프로토콜은 라우터 안에서 실행된다.

각 라우터는 다른 라우터와 정보를 주고받고, 그 결과로 자신의 포워딩 테이블을 만든다.

이 방식은 분산적이다.

네트워크 전체를 통제하는 하나의 중앙 장치가 아니라, 여러 라우터가 각자 라우팅 프로토콜을 실행하면서 전체 경로를 구성한다.

## 4.1.4 SDN 방식의 제어 평면

최근에는 **SDN(Software-Defined Networking)** 방식도 중요해졌다.

SDN에서는 제어 평면을 라우터 내부에 분산해두는 대신, 별도의 원격 컨트롤러가 관리할 수 있다.

라우터는 주로 데이터 평면 기능, 즉 패킷 포워딩을 수행한다.

경로 계산이나 정책 결정은 중앙화된 컨트롤러가 맡는다.

이 방식에서는 컨트롤러가 라우터의 포워딩 테이블을 직접 설정할 수 있다.

SDN의 장점은 네트워크를 프로그램처럼 제어할 수 있다는 점이다.

예를 들어 특정 트래픽을 특정 경로로 보내거나, 보안 정책을 중앙에서 일관되게 적용하는 것이 쉬워진다.

## 4.1.5 네트워크 서비스 모델

네트워크 계층은 트랜스포트 계층에게 어떤 서비스를 제공할 수 있을까?

이 질문을 **네트워크 서비스 모델(network service model)** 이라고 한다.

개념적으로는 다음과 같은 서비스를 생각할 수 있다.

### 보장된 전달

패킷이 목적지까지 반드시 도착하도록 보장하는 서비스이다.

### 지연 제한 보장

패킷이 일정 시간 안에 도착하도록 보장하는 서비스이다.

예를 들어 100ms 안에 도착해야 한다는 식의 보장이다.

### 순서 보장

패킷들이 보낸 순서대로 도착하도록 보장하는 서비스이다.

### 최소 대역폭 보장

송신자와 수신자 사이에 일정한 전송률을 보장하는 서비스이다.

### 보안 서비스

패킷의 기밀성이나 무결성을 보장하는 서비스도 생각할 수 있다.

## 4.1.6 인터넷의 네트워크 서비스 모델

인터넷의 IP는 매우 단순한 서비스 모델을 제공한다.

이를 **최선형 서비스(best-effort service)** 라고 한다.

최선형 서비스는 이름 그대로 최선을 다해 전달하려고 하지만, 강한 보장은 하지 않는다.

IP는 다음을 보장하지 않는다.

- 패킷 전달 보장
- 지연 시간 보장
- 순서 보장
- 대역폭 보장
- 중복 전달 방지

패킷은 손실될 수 있고, 순서가 바뀔 수 있고, 지연될 수 있다.

그럼에도 인터넷이 성공할 수 있었던 이유는 IP가 단순하기 때문이다.

네트워크 계층은 단순하게 유지하고, 필요한 신뢰성이나 제어 기능은 TCP 같은 상위 계층에서 제공한다.

이러한 설계는 인터넷을 확장 가능하게 만들었다.

## 정리

네트워크 계층은 송신 호스트에서 수신 호스트까지 데이터그램을 전달하는 역할을 한다.

포워딩은 라우터 내부에서 패킷을 입력 포트에서 출력 포트로 옮기는 작업이고, 라우팅은 출발지에서 목적지까지의 경로를 결정하는 작업이다.

데이터 평면은 포워딩을 담당하고, 제어 평면은 경로 계산과 포워딩 테이블 구성을 담당한다.

인터넷의 IP는 최선형 서비스를 제공한다.

즉, 네트워크 계층은 패킷 전달을 위해 최선을 다하지만, 도착 여부나 지연 시간 같은 강한 보장은 하지 않는다.
