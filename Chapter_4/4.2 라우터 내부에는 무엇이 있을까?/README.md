# 4.2 라우터 내부에는 무엇이 있을까?

이번 절에서는 라우터 내부 구조를 살펴본다.

라우터의 가장 중요한 역할은 입력 링크로 들어온 패킷을 적절한 출력 링크로 보내는 것이다.

겉으로 보면 단순히 패킷이 들어오고 나가는 장비처럼 보이지만, 내부에서는 여러 구성요소가 빠르게 협력한다.

라우터는 크게 다음 부분으로 구성된다.

1. 입력 포트
2. 스위칭 구조
3. 출력 포트
4. 라우팅 프로세서

## 4.2.1 입력 포트

입력 포트는 물리 링크로부터 패킷을 받아들이는 부분이다.

입력 포트에서는 다음 작업이 일어난다.

### 물리 계층 처리

입력 링크로 들어온 비트들을 수신한다.

즉, 전기 신호나 광 신호로 들어온 데이터를 비트 단위로 해석한다.

### 링크 계층 처리

이더넷 같은 링크 계층 프로토콜을 처리한다.

프레임 오류 검출, 프레임 추출 같은 작업이 이 단계에 포함된다.

### 포워딩 테이블 검색

입력 포트는 패킷의 목적지 주소를 보고 포워딩 테이블을 검색한다.

그 결과 패킷을 어느 출력 포트로 보내야 하는지 결정한다.

이 작업은 매우 빠르게 수행되어야 한다.

라우터가 초당 수많은 패킷을 처리하기 때문이다.

## 4.2.2 목적지 기반 포워딩

전통적인 IP 라우터는 주로 **목적지 기반 포워딩(destination-based forwarding)** 을 수행한다.

즉, 패킷의 목적지 IP 주소를 보고 출력 포트를 결정한다.

하지만 모든 IP 주소를 포워딩 테이블에 하나씩 저장할 수는 없다.

IPv4 주소만 해도 32비트이므로 가능한 주소 수가 매우 많다.

그래서 라우터는 주소 범위를 나타내는 프리픽스(prefix)를 사용한다.

예를 들어 다음과 같은 포워딩 테이블이 있다고 하자.

| 프리픽스 | 출력 링크 |
| --- | --- |
| 11001000 00010111 00010 | 0 |
| 11001000 00010111 00011000 | 1 |
| 11001000 00010111 00011 | 2 |
| otherwise | 3 |

라우터는 목적지 주소와 프리픽스를 비교해 가장 적절한 항목을 찾는다.

## 4.2.3 최장 프리픽스 매칭

하나의 목적지 주소가 여러 프리픽스와 동시에 매칭될 수 있다.

이때 라우터는 **최장 프리픽스 매칭(longest prefix matching)** 을 사용한다.

즉, 매칭되는 항목 중 가장 긴 프리픽스를 선택한다.

왜 가장 긴 프리픽스를 선택할까?

프리픽스가 길수록 더 구체적인 주소 범위를 의미하기 때문이다.

예를 들어 다음 두 항목이 있다고 하자.

```text
11001000 00010111 *
11001000 00010111 00011000 *
```

어떤 목적지 주소가 두 항목 모두와 매칭된다면, 두 번째 항목이 더 구체적이다.

따라서 라우터는 더 긴 프리픽스를 가진 두 번째 항목을 선택한다.

최장 프리픽스 매칭은 IP 라우터 포워딩에서 매우 중요한 규칙이다.

## 4.2.4 스위칭 구조

입력 포트에서 출력 포트가 결정되면, 패킷은 라우터 내부의 **스위칭 구조(switching fabric)** 를 지나 출력 포트로 이동한다.

스위칭 구조는 입력 포트와 출력 포트를 연결하는 라우터의 내부 통로이다.

대표적인 스위칭 방식은 다음 세 가지이다.

1. 메모리를 통한 스위칭
2. 버스를 통한 스위칭
3. 상호연결 네트워크를 통한 스위칭

### 메모리를 통한 스위칭

초기의 라우터는 입력 포트로 들어온 패킷을 라우팅 프로세서의 메모리로 복사했다.

그 후 다시 메모리에서 출력 포트로 복사했다.

이 방식은 구조가 단순하지만, 메모리 대역폭이 병목이 될 수 있다.

패킷 하나가 메모리에 한 번 들어가고 다시 나와야 하므로, 처리 속도가 메모리 속도에 크게 의존한다.

### 버스를 통한 스위칭

버스 방식에서는 입력 포트가 패킷을 공유 버스로 보낸다.

모든 출력 포트가 버스를 볼 수 있지만, 패킷에 붙은 내부 태그를 보고 자신에게 해당하는 패킷만 받아들인다.

이 방식은 메모리 방식보다 빠를 수 있지만, 한 번에 하나의 패킷만 버스를 사용할 수 있다.

따라서 버스 대역폭이 병목이 될 수 있다.

### 상호연결 네트워크를 통한 스위칭

상호연결 네트워크 방식은 여러 입력 포트와 출력 포트를 병렬로 연결할 수 있는 구조이다.

대표적으로 크로스바(crossbar) 스위치가 있다.

서로 다른 입력 포트의 패킷들이 서로 다른 출력 포트로 가는 경우, 동시에 여러 패킷을 전달할 수 있다.

고성능 라우터에서는 이런 방식이 널리 사용된다.

## 4.2.5 출력 포트

출력 포트는 스위칭 구조를 통해 도착한 패킷을 실제 출력 링크로 내보낸다.

출력 포트에서는 다음 작업이 일어난다.

1. 패킷 버퍼링
2. 스케줄링
3. 링크 계층 처리
4. 물리 계층 전송

출력 링크가 바쁘면 패킷은 출력 포트 큐에서 기다려야 한다.

큐가 가득 차면 새로 도착한 패킷이 버려질 수 있다.

이것이 라우터에서 패킷 손실이 발생하는 대표적인 이유 중 하나이다.

## 4.2.6 큐잉은 어디에서 발생할까?

라우터에서는 입력 포트와 출력 포트 양쪽에서 큐잉이 발생할 수 있다.

### 입력 포트 큐잉

입력 포트에서 패킷이 스위칭 구조로 들어가야 하는데, 스위칭 구조가 충분히 빠르지 않으면 입력 포트에 큐가 생긴다.

또는 여러 입력 포트가 같은 출력 포트로 패킷을 보내려고 하면 충돌이 발생할 수 있다.

이때 일부 패킷은 입력 포트에서 기다려야 한다.

### HOL 블로킹

입력 포트 큐잉에서 중요한 문제가 **HOL(Head-Of-the-Line) 블로킹** 이다.

큐의 맨 앞 패킷이 특정 출력 포트를 기다리고 있으면, 그 뒤에 있는 패킷들은 다른 출력 포트가 비어 있어도 앞으로 나갈 수 없다.

즉, 맨 앞 패킷 하나가 뒤 패킷들의 진행을 막는다.

이 현상은 입력 큐의 성능을 크게 떨어뜨릴 수 있다.

### 출력 포트 큐잉

스위칭 구조가 매우 빠르면 여러 입력 포트의 패킷이 짧은 시간에 같은 출력 포트로 몰릴 수 있다.

하지만 출력 링크는 한 번에 제한된 속도로만 패킷을 보낼 수 있다.

그래서 출력 포트에 큐가 생긴다.

출력 큐가 가득 차면 패킷 손실이 발생한다.

## 4.2.7 버퍼 크기

라우터의 큐에는 버퍼가 필요하다.

버퍼가 너무 작으면 일시적인 트래픽 증가에도 패킷 손실이 자주 발생한다.

반대로 버퍼가 너무 크면 큐잉 지연이 커질 수 있다.

전통적으로는 RTT와 링크 용량을 곱한 값 정도의 버퍼가 필요하다고 생각했다.

하지만 많은 TCP 흐름이 동시에 공유하는 고속 링크에서는 더 작은 버퍼로도 충분할 수 있다는 연구도 있다.

중요한 점은 버퍼가 손실만 줄이는 장치가 아니라 지연에도 영향을 준다는 것이다.

너무 큰 버퍼는 패킷을 오래 붙잡아 두어 지연을 키울 수 있다.

이를 흔히 버퍼블로트(bufferbloat) 문제라고 부른다.

## 4.2.8 패킷 스케줄링

출력 포트에 여러 패킷이 기다리고 있을 때, 라우터는 어떤 패킷을 먼저 보낼지 결정해야 한다.

이것을 **패킷 스케줄링(packet scheduling)** 이라고 한다.

대표적인 스케줄링 방식은 다음과 같다.

1. FIFO
2. 우선순위 큐잉
3. 라운드 로빈
4. WFQ

### FIFO

FIFO(First-In First-Out)는 먼저 도착한 패킷을 먼저 보내는 방식이다.

구현이 단순하지만, 모든 패킷을 동일하게 취급한다.

트래픽 종류에 따른 차별화가 어렵다.

### 우선순위 큐잉

우선순위 큐잉은 패킷을 우선순위별 큐에 넣고, 높은 우선순위 큐의 패킷을 먼저 보낸다.

실시간 음성이나 중요한 제어 트래픽을 먼저 처리할 수 있다.

하지만 낮은 우선순위 트래픽이 오래 기다리는 문제가 생길 수 있다.

### 라운드 로빈

라운드 로빈은 여러 큐를 순서대로 돌아가며 패킷을 전송한다.

각 클래스가 최소한의 전송 기회를 얻을 수 있다.

### WFQ

WFQ(Weighted Fair Queueing)는 각 클래스에 가중치를 부여해 전송 기회를 나누는 방식이다.

가중치가 큰 클래스는 더 많은 서비스 비율을 받는다.

단순한 라운드 로빈보다 더 정교한 자원 배분이 가능하다.

## 정리

라우터는 입력 포트, 스위칭 구조, 출력 포트, 라우팅 프로세서로 구성된다.

입력 포트에서는 패킷을 받고 포워딩 테이블을 검색한다.

스위칭 구조는 패킷을 입력 포트에서 출력 포트로 이동시킨다.

출력 포트에서는 패킷을 큐에 저장하고 링크로 내보낸다.

라우터 내부에서는 입력 큐와 출력 큐 모두에서 지연과 손실이 발생할 수 있다.

또한 패킷 스케줄링 방식에 따라 어떤 트래픽이 먼저 처리되는지가 달라진다.
